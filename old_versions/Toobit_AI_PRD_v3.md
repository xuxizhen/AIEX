## 一、产品概述

### 1.1 产品定义

**Toobit AI** 是一个以 AI 为核心驱动的加密货币交易 App。用户通过 Toobit 账户登录，即可获得 AI 辅助的全链路交易体验——AI 帮你看懂市场、辅助决策、跟上高手。

### 1.2 核心差异化

| 维度 | 传统交易所 | Toobit AI |
|------|----------|-----------|
| 核心价值 | 提供交易工具 | 提供交易结果和决策辅助 |
| 用户门槛 | 需要自己分析判断 | AI 降低分析门槛 |
| AI 角色 | 无 或 仅作为 chatbot | AI 贯穿全链路交易 |
| 跟单体系 | 仅支持跟真人 | 真人 + AI 混合跟单 |
| 商业模式 | 手续费 | 手续费 + 跟单分润 + AI 策略订阅 |

### 1.2.1 竞品对标分析

> 基于对 Cryptohopper、3Commas、Bitsgap、Pionex、Zignaly 等 14+ AI 交易平台的调研，市场呈现以下特征：

| 竞品类型 | 代表产品 | 商业模式 | 与我们的差异 |
|---------|---------|---------|----------|
| 第三方Bot工具 | Cryptohopper, 3Commas | 订阅制（$29-$129/月） | 需用户自配API Key，延迟高，无交易所级风控 |
| 内置Bot交易所 | Pionex | 免费Bot + 交易手续费 | Bot免费但无AI智能，功能固定 |
| 社交跟单平台 | Zignaly, WunderTrading | 收益分成 | 仅真人跟单，无AI策略 |
| 已关停项目 | Dash2Trade, 3EX, HaasOnline | 各异 | 缺乏交易所支撑，无法持续运营 |

**核心洞察**：多个竞品（Dash2Trade、3EX、HaasOnline）已在2024-2025年关停，市场正处于洗牌期。Toobit AI 作为交易所内部孵化的AI交易App，拥有账户直连、最低延迟、统一风控三大结构性优势，是第三方工具无法复制的。

### 1.3 产品核心亮点

| # | 亮点 | 说明 |
|---|------|------|
| 1 | AI 策略实盘跟单 | 市面首个支持像跟真人一样跟 AI 策略的 Crypto App，AI 策略有真实持仓、真实盈亏，完全透明 |
| 2 | AI 原生交易体验 | AI 嵌入每个交易环节：行情页 AI 简评、下单页 AI 风险提示、持仓页 AI 健康度评分 |
| 3 | 真人 + AI 混合跟单 | 跟单广场中真人交易员和 AI 策略统一展示、统一排行，用户自由选择 |
| 4 | 渐进式 AI 授权 | 用户可自主控制 AI 权限：只看→辅助确认→半自动→全托管 |
| 5 | 社交化交易网络 | 用户可成为交易员被关注和跟单，形成 Crypto 交易社区 |

### 1.4 产品愿景

- **短期（6个月）**：成为 Crypto 领域体验最好的 AI 辅助交易 App
- **中期（1年）**：建立最大的 AI 策略 + 真人交易员跟单生态
- **长期（2年）**：扩展多交易所 API 接入，成为跨平台 AI 交易入口

## 二、账户体系

### 2.1 初期：Toobit 账户直连

用户用 Toobit 账号登录后，App 自动通过 Toobit 内部 API 获得以下权限：

| 权限类型 | 说明 | API 接口 |
|---------|------|---------|
| 读取资产 | 现货余额、合约保证金、总资产 | `GET /api/v1/account/balance` |
| 查询行情 | 实时价格、K线、深度数据 | `GET /api/v1/market/ticker` |
| 查询订单 | 历史成交、当前委托 | `GET /api/v1/order/history` |
| 现货下单 | 限价/市价/条件单 | `POST /api/v1/spot/order` |
| 合约下单 | 开多/开空/平仓 | `POST /api/v1/futures/order` |
| 合约持仓 | 查询合约持仓和保证金 | `GET /api/v1/futures/position` |

**零门槛体验**：用户无需手动配置 API Key，Toobit 登录后后台自动创建子账号 API 并绑定到 App。

### 2.2 未来扩展（本期不做）

支持绑定其他交易所 API Key（OKX、Binance、Bybit 等），实现跨交易所的统一资产视图和交易能力。

## 三、App 信息架构

### 3.1 底部 Tab 结构

```
┌─────────────────────────────────────────────┐
│                                             │
│              [页面内容区域]                    │
│                                             │
├─────┬─────┬─────┬─────┬─────┤
```

| Tab | 名称 | 核心职责 | 优先级 |
|-----|------|---------|--------|

- **AI 浮动按钮**：App 右上角常驻 AI 入口（类似 RockFlow 的 Bobby 图标），任意页面可唤起 AI 对话

- **全局搜索**：支持搜索币种、交易员、AI 策略

#### 4.1.2 功能需求

**FR-1.1 总资产卡片**

| 字段 | 说明 | 数据来源 |
|------|------|---------|
| 总资产 | 现货 + 合约保证金 + 未实现盈亏，USD 计价 | Toobit API |
| 今日盈亏 | 今日0点至当前的资产变化 | 计算值 |
| 资产趋势 | 近7天的迷你折线图 | 历史快照 |

点击跳转：→ 资产详情页

**FR-1.2 AI 每日热点**

| 字段 | 说明 |
|------|------|
| 标题 | AI 生成的热点标题（≤30字） |
| 简评 | AI 的 2-3 句话解读 |
| 相关币种 | 标签形式显示关联币种 |
| AI 观点 | 看涨（绿色）/ 看跌（红色）/ 观望（灰色） |
| 操作入口 | 「交易」按钮 → 跳转到交易页 |

- 每日自动生成 3-6 条，每 4 小时刷新
- 数据源：行情变动 + 链上异动 + 新闻舆情 + 技术指标信号
- 用户可点赞/收藏

**FR-1.3 热门币种涨跌榜**

- 三个子 Tab：24h 涨幅榜 / 24h 跌幅榜 / 24h 成交量榜
- 每榜展示 Top 10
- 每条显示：币种图标、名称、代码、当前价格、涨跌幅（颜色区分）、迷你24h K线
- 点击 → 跳转到该币种交易页

**FR-1.4 AI 策略精选推荐**

- 横向滑动卡片，展示 2-3 个近期表现好的 AI 策略
- 每个卡片显示：策略名称、近 7 天/30 天收益率、运行天数、跟单人数
- CTA「跟单」按钮 → 跳转到跟单广场对应策略详情

**FR-1.5 投教内容区**

- 运营配置的投教内容（短视频 / 图文）
- 分类：新手入门、交易技巧、AI 功能教程
- 支持标记"已读"

**FR-3.1 AI 一句话下单**

用户在 AI 输入框中输入自然语言，AI 解析后自动填充订单参数：

| 用户输入 | AI 解析结果 | 自动填充 |
|---------|-----------|---------|

## 五、跟单分润详细设计

### 5.1 真人交易员分润模型

#### 5.1.1 分润计算公式

```
单笔分润 = max(0, 跟单头寸盈利 - 高水位亏损填补) × 分润比例
```

#### 5.1.2 分润比例体系

| 交易员等级 | 升级条件 | 交易员分润 | 平台抽成 | 跟单者获得 |
|---------|---------|---------|---------|---------|
| 🥉 青铜 | 30天收益>0%, 跟单者≥5 | 8% | 2% | 90% |
| 🥈 白银 | 60天收益>10%, 跟单者≥20 | 9% | 2% | 89% |
| 🥇 黄金 | 90天收益>20%, 跟单者≥50, 回撤<15% | 10% | 2.5% | 87.5% |
| 💎 钻石 | 180天收益>40%, 跟单者≥100, 回撤<10% | 12% | 3% | 85% |

#### 5.1.3 高水位线机制 (High-Water Mark)

**原理**：如果跟单者之前有亏损，交易员需要先帮用户回本，才能对新增盈利收取分润。

**计算示例**：

| 交易序号 | 盈亏 | 累计盈亏 | 高水位线 | 可分润金额 | 交易员获得(10%) |
|---------|------|---------|---------|---------|-------------|
| #1 | +100U | +100U | 100U | 100U | 10U |
| #2 | -60U | +40U | 100U | 0（未超高水位） | 0 |
| #3 | +80U | +120U | 120U | 20U（仅超出高水位的部分） | 2U |
| #4 | +50U | +170U | 170U | 50U | 5U |

#### 5.1.4 结算规则

| 规则 | 说明 |
|------|------|
| 结算时机 | 每笔跟单头寸平仓后实时结算 |
| 最小结算金额 | 分润 ≥ 0.1 USDT 才结算，否则累积到下次 |
| 结算币种 | USDT |
| 到账方式 | 自动到交易员的 Toobit 账户余额 |
| 查看明细 | 「我的」→ 「跟单管理」→ 「分润收入记录」 |

## 六、渐进式 AI 授权体系

| 等级 | 名称 | AI 能做什么 | 不能做什么 | 适用场景 |
|------|------|-----------|---------|---------|
| L1 | 只看不做 | 行情分析、持仓诊断、市场扫描、知识问答 | 任何涉及账户操作的行为 | 新用户 / 极度谨慎型 |
| L2 | 辅助确认 | L1全部 + 生成订单预览 + 生成跟单方案 | 必须用户点击确认才执行 | **大部分用户（默认）** |
| L3 | 半自动 | L2全部 + 自动执行止损止盈 + 自动调整跟单参数 | 开新仓/大额操作仍需确认 | 有经验用户 |
| L4 | 全托管 | AI策略跟单完全自动化 | 仅在跟单范围内操作 | 跟单用户 |

**切换入口**：「我的」→「AI偏好设置」→「AI授权等级」

**安全机制**：
- 升级到 L3/L4 需要二次验证（密码或生物识别）
- L3/L4 设置单日最大自动交易金额上限
- 任何等级下，用户都可以随时一键暂停所有 AI 操作

## 七、红线清单

| # | 红线 | 实现方式 |
|---|------|---------|
| 1 | AI 绝不能未经确认自动交易 | L1-L2 强制确认；L3-L4 仅在预设规则范围内自动化 |
| 2 | 每个AI建议必须附带理由 | 所有AI输出包含"为什么"字段 |
| 3 | 不能承诺收益 | 全局禁用"稳赚""保本"等词汇；AI策略展示必含风险提示 |
| 4 | 不能泄露用户数据 | 交易员公开记录脱敏（不显示金额）；AI 不跨用户共享数据 |
| 5 | 不能制造 FOMO | 推送频率可控；不发送"错过就亏了"类文案 |
| 6 | 不能诱导高频交易 | 检测到频繁交易时弹出冷静提示 |
| 7 | 免责声明清晰 | 所有AI建议底部附"仅供参考，投资风险自担" |
